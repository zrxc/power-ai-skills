# 高级搜索技能
用于生成符合企业列表页模式的高级搜索区域和查询状态管理。

## 适用场景

- 查询条件较多的列表页
- 需要折叠 / 展开搜索区域
- 搜索项按条件动态显隐
- 搜索区与列表、统计卡片、图表联动

## 按需读取的参考文件

- 搜索区模板：`references/templates.md`
- 搜索状态规则：`references/rules.md`

## 必须遵循的项目模式

### 结构

- 搜索条件单独维护，不与表格数据混放
- 搜索区优先复用 `pc-advanced-search` 或企业现有搜索模式
- 重置时恢复默认条件并重置页码

### 状态

- 查询条件对象命名稳定，例如 `searchFormDatas`
- 页面初始化时明确默认搜索条件
- 筛选变化后集中触发数据刷新

### 联动

- 搜索区与分页联动时，搜索后页码回到 `1`
- 搜索区与图表联动时，统一入口触发刷新

## 实现清单

1. 明确搜索项和默认值
2. 实现搜索、重置、展开收起逻辑
3. 将搜索条件与列表 / 图表查询解耦
4. 保持搜索区配置和请求参数映射清晰

## 何时切换到其他技能

- 当页面主体其实是标准列表页，而高级搜索只是附属区域时，切换到 `basic-list-page`
- 当问题已经从查询条件组织转成统计图表联动与看板布局时，切换到 `chart-dashboard`
- 当阻塞点变成字段校验、表单项交互和复杂输入规则时，补充 `form-skill`
- 当当前问题落在导出查询结果、批量下载或查询后文件处理时，补充 `upload-import-export`

## 注意事项

- 不要把请求参数转换逻辑散落在模板里
- 搜索项过多时优先折叠和分组
- 日期范围、枚举值要统一转换规则
