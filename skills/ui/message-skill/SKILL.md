# 消息反馈技能
用于生成符合项目约定的操作反馈和确认提示流程。

## 适用场景

- 新增、编辑、删除成功提示
- 接口失败提示
- 删除前确认框
- 表单校验失败提示
- 批量操作提示

## 按需读取的参考文件

- 成功失败提示示例：`references/templates.md`
- 删除确认示例：`references/templates.md`
- 批量操作示例：`references/templates.md`

## 必须遵循的项目模式

### 提示类型

- 成功操作使用 `ElMessage.success`
- 失败操作使用 `ElMessage.error`
- 前置提醒使用 `ElMessage.warning`
- 需要用户确认时使用 `ElMessageBox.confirm`

### 行为约定

- 删除类动作先确认，再请求
- 用户取消确认时不提示错误
- 表单未通过校验时给出明确引导

## 实现清单

1. 根据动作类型选择合适的提示组件
2. 为成功和失败分别补充提示文本
3. 危险操作加入二次确认
4. 对用户取消行为与真实异常做区分

## 常见提示文案

- 新增成功
- 编辑成功
- 删除成功
- 保存失败，请重试
- 请先选择要操作的数据

## 何时切换到其他技能

- 当问题首先是页面骨架和列表交互，而不是反馈提示时，切换到 `basic-list-page` 或 `tree-list-page`
- 当主要复杂度在表单字段与校验规则，而不是校验失败后的提示方式时，切换到 `form-skill`
- 当任务重点是弹窗状态流，而不是确认和反馈文案时，切换到 `dialog-skill`
- 当阻塞点变成接口错误结构、统一请求拦截或全局异常处理时，切换到 `api-skill` 或 `runtime-extension-skill`

## 注意事项

- 提示文案保持简洁直接
- 不要把用户取消误判成异常失败
- 批量操作前先校验选中项数量
