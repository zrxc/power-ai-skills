# 图表看板技能
用于生成基于 ECharts 和企业页面容器的看板类页面。

## 适用场景

- 概览看板
- 告警统计
- 资源监控趋势
- 卡片 + 图表混合布局
- 图表弹窗详情页

## 按需读取的参考文件

- 看板页面模板：`references/templates.md`
- 图表组织方式：`references/charts.md`

## 必须遵循的项目模式

### 布局

- 页面按卡片区、筛选区、图表区拆分
- 图表模块支持 `loading`、空状态和错误态
- 弹窗详情仍优先使用 `pc-dialog`

### 数据

- 图表数据和列表数据分开管理
- 查询条件变化时集中刷新图表
- 图表 `option` 构造逻辑与接口请求逻辑分离

### 组件

- 状态卡片尽量抽成子组件
- 图表实例销毁与重建逻辑明确
- 复杂看板避免把所有 `option` 堆到一个文件中

## 实现清单

1. 划分看板模块和筛选项
2. 定义图表数据请求和 `option` 构造方法
3. 实现卡片、图表、表格等子模块
4. 补充空状态、错误态和刷新逻辑

## 何时切换到其他技能

- 当主体其实是标准列表页或查询列表，而图表只占很小一部分时，切换到 `basic-list-page`
- 当核心复杂度在高级筛选条件组织和查询状态联动，而不是图表展示本身时，补充 `advanced-search-skill`
- 当需要详情弹窗、图表 drill-down 或操作确认链路时，补充 `dialog-skill`
- 当阻塞点变成接口契约、聚合统计字段或返回结构定义时，切换到 `api-skill`

## 注意事项

- 图表 `option` 与业务请求分层
- 大看板要避免单文件过长
- 图表组件销毁时清理实例和监听器
