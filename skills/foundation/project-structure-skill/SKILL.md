# 项目结构技能
用于统一企业前端项目的目录划分、职责边界和代码落位规则。

## 适用场景

- 新建项目
- 大模块拆分
- 老项目目录治理
- 统一页面、接口、组件、store 归档规则

## 按需读取的参考文件

- 标准目录模板：`references/structure.md`
- 模块拆分原则：`references/rules.md`

## 必须遵循的项目模式

- 页面放 `src/views`
- 接口放 `src/api/modules`
- 项目私有组件放 `src/components`
- store 放 `src/store`
- 工具函数放 `src/utils`

## 实现清单

1. 明确模块边界
2. 确认页面、接口、组件、状态、工具的归属目录
3. 将公共能力与业务私有能力拆开
4. 保持命名与目录粒度一致

## 何时切换到其他技能

- 当问题只是在确认项目基础技术基线，而不是目录落位时，切换到 `power-foundation-app`
- 当当前任务的关键是企业组件选型，而不是代码归档位置时，切换到 `power-component-library`
- 当问题已经进入具体页面实现，例如标准列表或树表联动时，切换到对应页面 skill
- 当阻塞点变成运行时扩展、全局 store 或请求能力，而不是目录结构时，切换到 `runtime-extension-skill`

## 注意事项

- 不要把业务页面和通用组件混放
- 不要把接口函数写进页面文件
- 保持目录层级不过深
