# 路由菜单技能
用于统一企业前端项目中的菜单、路由和模块挂载规则。

## 适用场景

- 新增一个业务菜单模块
- 新增路由页面并接入菜单
- 子系统页面接入
- 菜单显隐和顺序调整

## 按需读取的参考文件

- 路由注册模板：`references/templates.md`
- 规则说明：`references/rules.md`

## 必须遵循的项目模式

### 路由

- 页面新增后明确对应路由和名称
- 子模块路由按业务域拆分
- 路由命名保持稳定且可检索

### 菜单

- 菜单、页面、权限资源保持对应关系
- 隐藏菜单与隐藏页面区分处理
- 页面标题、图标、排序等元信息统一管理

## 实现清单

1. 明确新页面归属模块
2. 添加路由定义和页面组件映射
3. 补充菜单元信息和权限资源映射
4. 校验隐藏菜单、详情页、弹窗页等特殊路由

## 何时切换到其他技能

- 当主要问题是在讨论项目目录结构，而不是菜单和路由映射时，切换到 `project-structure-skill`
- 当核心复杂度已经转成页面权限、按钮权限和资源级裁剪时，切换到 `permission-page`
- 当任务重点是某个列表页、树表页或详情页实现，而菜单接入只是附属工作时，切换到对应页面 skill
- 当阻塞点上升到运行时路由能力、动态注册机制或全局守卫时，补充 `runtime-extension-skill`

## 注意事项

- 页面路径、路由名、菜单标识尽量统一
- 不要在多个位置重复定义同一菜单逻辑
