# 全局状态技能
用于统一企业前端项目中全局 store 和运行时全局数据的使用方式。

## 适用场景

- 获取用户信息、token、前端配置
- 使用 `globalStore`
- 管理跨页面共享状态
- 组织领域 store 模块

## 按需读取的参考文件

- 全局 store 示例：`references/templates.md`
- 使用规则：`references/rules.md`

## 必须遵循的项目模式

### 数据来源

- 优先复用 `@power/runtime-vue3` 已有全局 store 能力
- 用户信息、token、系统配置不要在业务侧重复封装

### 组织方式

- 全局状态与页面局部状态分离
- 领域 store 按模块拆分
- 避免把一次性页面状态提升为全局状态

## 实现清单

1. 判断数据是否属于真正全局共享
2. 明确来源于运行时全局 store 还是项目私有 store
3. 抽离通用 getter、setter 或 hooks
4. 保持依赖方向清晰

## 何时切换到其他技能

- 当主要问题是在讨论项目目录结构和模块归档，而不是状态提升边界时，切换到 `project-structure-skill`
- 当任务核心已经上升到运行时层全局行为、router、hooks 或统一请求能力时，切换到 `runtime-extension-skill`
- 当当前问题只是页面局部状态、表单状态或弹窗状态管理，而不是全局共享数据时，切换到对应页面或表单 skill
- 当阻塞点变成项目基础上下文和技术基线，而不是 store 组织方式时，切换到 `power-foundation-app`

## 注意事项

- 不要滥用全局状态
- 全局配置读取逻辑统一封装
- 页面退出后无意义状态不要常驻全局
