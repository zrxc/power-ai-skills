# 槽位信息抽取

## 目标

从自然语言中抽取页面实现所需的关键信息，让用户不必手动点名 skill。

## 抽取前提

- 先参考 `aliases.md` 做口语归一
- 再按本文件抽取槽位
- 权限点和步骤表单由专门 reference 继续细化

## 需要抽取的信息

- `业务对象`
- `树对象`
- `列表对象`
- `字段`
- `操作`
- `批量操作`
- `行级交互`
- `表单模式`
- `弹层形式`
- `附加能力`
- `视图切换`

## 抽取规则

### 业务对象

触发短语：

- `用户列表`
- `角色清单`
- `设备台账`
- `工单目录`

抽取结果：

- `用户`
- `角色`
- `设备`
- `工单`

### 树对象

触发短语：

- `树使用部门`
- `左边组织树`
- `左侧部门右侧用户`
- `按区域树筛选`

抽取结果：

- `部门`
- `组织`
- `区域`

### 列表对象

触发短语：

- `点击部门后显示用户`
- `点击部门显示用户`
- `左侧部门右侧用户`
- `右边列表显示设备`
- `树节点切换后刷新角色列表`

抽取结果：

- `用户`
- `设备`
- `角色`

### 字段

触发短语：

- `字段有用户名、邮箱、手机号、状态`
- `列表字段包含名称、编码、状态、创建时间`
- `表头展示部门名称、负责人、更新时间`

抽取结果：

- 保留字段顺序
- 默认把顿号、逗号、中文逗号都视作字段分隔符

### 操作

触发短语：

- `支持新增、编辑、删除`
- `支持增删改`
- `支持查看、导出`
- `支持创建、维护、作废`

抽取结果：

- 统一映射到标准动作：`新增`、`编辑`、`删除`、`查看`、`导入`、`导出`

### 批量操作

触发短语：

- `支持批量删除、批量导出`
- `支持批量启用、批量停用`
- `支持勾选后统一处理`

抽取结果：

- `批量删除`
- `批量导出`
- `批量启用`
- `批量停用`

### 行级交互

触发短语：

- `状态支持行内开关`
- `支持行内编辑`
- `列表状态可直接切换`

抽取结果：

- `行内开关`
- `行内编辑`
- `状态切换`

### 表单模式

触发短语与结果：

- `使用表单设计器` -> `form-designer-skill`
- `动态表单` -> `form-designer-skill`
- `步骤表单` / `向导表单` -> `步骤表单`
- 未提及 -> 默认 `form-skill`

### 弹层形式

触发短语与结果：

- `新增编辑用弹窗` -> `弹窗`
- `新增编辑走抽屉` -> `抽屉`
- `新增编辑在独立页面维护` -> `独立页面`

### 视图切换

触发短语与结果：

- `顶部 tabs 切换不同状态列表` -> `tabs`
- `按状态切换列表` -> `状态切换`
- `上面基本信息下面角色列表` -> `主子表`

### 附加能力

触发短语与结果：

- `高级搜索` -> `advanced-search-skill`
- `导入导出` -> `upload-import-export`
- `权限` -> `permission-page`
- `预览` -> `file-preview-skill`
- `图表` / `统计卡片` -> `chart-dashboard`

## 与其他 reference 的分工

- 若操作后带括号权限点，例如 `新增（user_add）`，继续读 `permission-extraction.md`
- 若出现 `第一步 / 第二步 / 第三步`，继续读 `step-form-extraction.md`

## 典型抽取结果

输入：

```text
新增一个用户列表，字段有用户名、邮箱、手机号、状态，支持新增、编辑、删除，使用表单设计器
```

抽取结果：

- 骨架：普通列表页
- 业务对象：用户
- 字段：用户名、邮箱、手机号、状态
- 操作：新增、编辑、删除
- 表单模式：表单设计器

输入：

```text
新增一个用户树列表，树使用部门，点击部门后显示用户列表，字段有用户名、邮箱、手机号、状态，支持新增、编辑、删除
```

抽取结果：

- 骨架：树列表页
- 树对象：部门
- 列表对象：用户
- 字段：用户名、邮箱、手机号、状态
- 操作：新增、编辑、删除
- 表单模式：普通表单

输入：

```text
添加一个用户管理列表，字段有用户名、邮箱、手机号、状态，支持新增（user_add）、编辑（user_edit）、删除（user_delete），新增表单分三步，第一步选择部门，第二步添加用户，第三步添加角色
```

抽取结果：

- 骨架：普通列表页
- 业务对象：用户
- 字段：用户名、邮箱、手机号、状态
- 操作：新增、编辑、删除
- 表单模式：步骤表单
- 权限点抽取：交给 `permission-extraction.md`
- 步骤抽取：交给 `step-form-extraction.md`
