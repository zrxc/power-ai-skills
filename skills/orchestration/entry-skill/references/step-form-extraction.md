# 步骤表单抽取
## 目标

从自然语言里抽取多步骤表单或向导式流程，识别步骤数量、步骤名称和每一步职责。

## 适用句式

- `新增表单分三步，第一步选择部门，第二步添加用户，第三步添加角色`
- `这是一个步骤表单`
- `新增使用向导表单`

## 识别规则

### 表单模式识别

命中以下关键词时，视为步骤表单：

- `步骤表单`
- `分步骤表单`
- `向导表单`
- `分三步`
- `第一步`

### 步骤抽取

将以下内容抽成步骤数组：

- `第一步选择部门`
- `第二步添加用户`
- `第三步添加角色`

输出为：

- `step1: 选择部门`
- `step2: 添加用户`
- `step3: 添加角色`

### 默认理解

- 步骤表单默认仍以 `form-skill` 为主
- 若用户同时提到 `表单设计器` / `动态配置`，再切换为 `form-designer-skill`
- 步骤表单通常和 `dialog-skill` 同时出现，除非用户明确要求独立页面

## 命中后的默认补充

- 若场景是列表页里的新增/编辑
  - 保持原页面骨架 skill
  - 补 `dialog-skill`
  - 补 `form-skill`
- 若同时提到 `表单设计器`
  - 用 `form-designer-skill` 替代 `form-skill`

## 示例

输入：

```text
添加一个用户管理列表，字段有用户名、邮箱、手机号、状态，支持新增、编辑、删除，新增表单分三步，第一步选择部门，第二步添加用户，第三步添加角色
```

抽取结果：

- 表单模式：步骤表单
- 步骤：
  - 第一步：选择部门
  - 第二步：添加用户
  - 第三步：添加角色

- 默认补充：
  - `dialog-skill`
  - `form-skill`
