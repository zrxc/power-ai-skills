# 通用场景类型目录

## 目标
- 从企业真实项目中反推可复用的通用前端场景类型。
- 不绑定具体业务名词，业务对象只作为槽位输入。
- 让用户可以只说“做一个列表”“做一个树列表”“做一个详情页”，再补字段、操作、权限、步骤等要素。

## 一、核心原则

### 1. 先识别页面骨架，再识别业务对象
- 不先问“这是用户管理还是设备管理”。
- 先判断它属于哪种页面骨架：
  - 普通列表页
  - 树列表页
  - 详情页
  - 主子表详情页
  - 弹窗/抽屉主导页
  - 工作台/看板页
  - 设计器/画布页

### 2. 业务对象只作为槽位
- 例如：
  - 用户
  - 角色
  - 部门
  - 设备
  - 资源
  - 指标
  - 插件
  - 工单
  - 策略
- 这些不决定 skill 类型，只决定页面语义、字段、接口命名和文案。

### 3. 通用能力作为增强项拼装
- 不把“批量删除”“导入导出”“按钮权限”“步骤表单”当成独立页面骨架。
- 它们是附加能力，识别后自动拼到主骨架上。

## 二、通用页面骨架

### 1. 普通列表页
- 典型结构：
  - 查询区
  - 表格区
  - 分页
  - 新增/编辑/删除
- 用户表达特征：
  - 列表
  - 管理页
  - 表格
  - 支持增删改查
- 通用句式：
  - 新增一个{对象}列表，字段有{字段列表}，支持新增、编辑、删除。
  - 做一个{对象}管理页面，带搜索和分页。
- 默认映射：
  - `basic-list-page`
  - `dialog-skill`
  - `api-skill`
  - `message-skill`

### 2. 树列表页
- 典型结构：
  - 左树
  - 右表
  - 点击树节点刷新右侧列表
- 用户表达特征：
  - 树列表
  - 左树右表
  - 树筛选列表
- 通用句式：
  - 新增一个{对象}树列表，树使用{树对象}，点击{树对象}后显示{对象}列表。
  - 做一个左树右表页面，左侧是{树对象}树，右侧是{对象}列表。
- 默认映射：
  - `tree-list-page`
  - `dialog-skill`
  - `api-skill`
  - `message-skill`

### 3. 详情页
- 典型结构：
  - 基础信息
  - tabs 区
  - 附加列表 / 附件 / 图表
- 用户表达特征：
  - 详情页
  - 查看详情
  - 基本信息 + 下方明细
- 通用句式：
  - 做一个{对象}详情页，上面是基本信息，下面是{子区域1}和{子区域2}。
  - 做一个{对象}详情查看页，支持查看附件和历史记录。
- 默认映射：
  - `detail-page-skill`
  - `api-skill`
  - `message-skill`

### 4. 主子表详情页
- 典型结构：
  - 主信息卡片
  - 一个或多个子表
  - 可能带附件或预览
- 用户表达特征：
  - 主子表
  - 上面详情下面列表
  - 基本信息 + 子记录
- 通用句式：
  - 做一个{对象}详情页，上面是基本信息，下面是{子对象}列表。
  - 做一个{对象}详情页，包含{子对象1}列表和{子对象2}列表。
- 默认映射：
  - `detail-page-skill`
  - `basic-list-page`
  - `api-skill`

### 5. 弹窗/抽屉主导页
- 典型结构：
  - 页面主体是列表
  - 新增编辑通过弹窗或抽屉完成
- 用户表达特征：
  - 新增弹窗
  - 编辑抽屉
  - 抽屉表单
- 通用句式：
  - 做一个{对象}列表，新增编辑使用弹窗。
  - 做一个{对象}列表，编辑用抽屉。
- 默认映射：
  - `dialog-skill`
  - `form-skill`

### 6. 工作台/看板页
- 典型结构：
  - 多卡片组合
  - 图表
  - 概览区
  - 快捷入口
- 用户表达特征：
  - 工作台
  - 看板
  - 概览
  - dashboard
- 通用句式：
  - 做一个{对象}工作台，展示概览卡片、趋势图和快捷入口。
  - 做一个看板页，包含统计卡片、图表和列表。
- 默认映射：
  - `chart-dashboard`
  - `detail-page-skill`

### 7. 设计器/画布页
- 典型结构：
  - 左侧配置
  - 中间画布
  - 右侧属性
  - 实时预览
- 用户表达特征：
  - 设计器
  - 画布
  - 拖拽
  - 配置器
- 通用句式：
  - 做一个{对象}设计器，支持拖拽配置和实时预览。
  - 做一个流程设计页，支持节点配置和连线配置。
- 默认映射：
  - `form-designer-skill`

## 三、通用增强能力

### 1. 基础 CRUD
- 触发词：
  - 新增
  - 编辑
  - 删除
  - 查看
- 说明：
  - 出现 CRUD 时，默认补 `dialog-skill`、`api-skill`、`message-skill`。

### 2. 搜索能力
- 触发词：
  - 搜索
  - 筛选
  - 高级搜索
- 说明：
  - 普通搜索归入主页面骨架。
  - 高级搜索自动补 `advanced-search-skill`。

### 3. 批量操作
- 触发词：
  - 批量删除
  - 批量导出
  - 批量导入
  - 批量启用
  - 批量停用
- 说明：
  - 导入导出自动补 `upload-import-export`。

### 4. 行内交互
- 触发词：
  - 行内开关
  - 行内编辑
  - 行内按钮
  - 操作列
- 说明：
  - 属于列表页增强项，不改变主骨架。

### 5. tabs 视图切换
- 触发词：
  - tabs
  - 顶部切换
  - 全部/启用/停用
  - 按状态切换
- 说明：
  - 属于列表或详情页的视图增强项。

### 6. 权限能力
- 触发词：
  - 新增（xxx_add）
  - 编辑（xxx_edit）
  - 删除（xxx_delete）
  - 查看（xxx_view）
- 说明：
  - 自动补 `permission-page`。
  - 同时抽取“操作 -> 权限点”映射。

### 7. 表单模式
- 类型：
  - 普通表单
  - 表单设计器
  - 步骤表单
  - 抽屉表单
- 说明：
  - 提到“表单设计器”时切 `form-designer-skill`。
  - 提到“第一步/第二步/第三步”时识别为步骤表单。

### 8. 附件与预览
- 触发词：
  - 附件
  - 文件预览
  - 图片预览
  - PDF 预览
- 说明：
  - 自动补 `file-preview-skill`。

### 9. 图表与统计
- 触发词：
  - 趋势图
  - 统计图
  - 饼图
  - 柱状图
  - 看板
- 说明：
  - 自动补 `chart-dashboard`。

## 四、通用槽位模型

### 1. 主对象
- 示例：
  - 用户
  - 角色
  - 设备
  - 资源
  - 任务

### 2. 树对象
- 示例：
  - 部门
  - 分类
  - 目录
  - 资源树

### 3. 字段列表
- 示例表达：
  - 字段有用户名、邮箱、手机号、状态
  - 列包含名称、编码、类型、创建时间

### 4. 操作列表
- 示例表达：
  - 支持新增、编辑、删除
  - 支持查看详情、导出、启停

### 5. 表单模式
- 示例表达：
  - 使用表单设计器
  - 新增表单分三步
  - 编辑使用抽屉

### 6. 权限点
- 示例表达：
  - 新增（user_add）、编辑（user_edit）、删除（user_delete）

### 7. 子对象
- 示例表达：
  - 详情页下面是角色列表和附件列表
  - 主信息下面是日志列表和告警列表

## 五、推荐的自然语言输入模板

### 1. 普通列表
```text
新增一个{对象}列表，字段有{字段列表}，支持新增、编辑、删除。
```

### 2. 普通列表 + 表单设计器
```text
新增一个{对象}列表，字段有{字段列表}，支持新增、编辑、删除，使用表单设计器。
```

### 3. 普通列表 + 权限点
```text
新增一个{对象}列表，字段有{字段列表}，支持新增（{add_perm}）、编辑（{edit_perm}）、删除（{delete_perm}）。
```

### 4. 普通列表 + 步骤表单
```text
新增一个{对象}列表，字段有{字段列表}，支持新增、编辑、删除，新增表单分三步，第一步{step1}，第二步{step2}，第三步{step3}。
```

### 5. 树列表
```text
新增一个{对象}树列表，树使用{树对象}，点击{树对象}后显示{对象}列表，字段有{字段列表}，支持新增、编辑、删除。
```

### 6. 树列表 + 树节点维护
```text
做一个{树对象}{对象}树列表，左侧{树对象}树支持新增、编辑、删除，右侧显示{对象}列表。
```

### 7. 详情页
```text
做一个{对象}详情页，上面是基本信息，下面是{子区域1}和{子区域2}。
```

### 8. 看板页
```text
做一个{对象}工作台，展示概览卡片、统计图和快捷入口。
```

## 六、从真实项目沉淀出来的结论
- 真正稳定复用的不是“监控对象”“告警规则”“工单”这些业务名词。
- 稳定复用的是：
  - 列表页
  - 树列表页
  - 详情页
  - 步骤表单
  - tabs 切换
  - 批量操作
  - 权限点映射
  - 工作台看板
  - 设计器画布
- 因此入口层应该优先维护“场景骨架 + 槽位提取 + 增强能力”，而不是维护越来越长的业务场景词典。
