# 权限页面技能
用于生成符合企业前端权限体系的页面、操作按钮和资源控制逻辑。

## 适用场景

- 页面需要按角色或资源进行显隐控制
- 列表页按钮需要按权限控制是否显示
- 表单或弹窗内字段需要按权限控制可编辑状态
- 页面初始化需要做权限校验和兜底处理

## 按需读取的参考文件

- 页面级权限模板：`references/templates.md`
- 按钮级权限模板：`references/templates.md`
- 接入原则：`references/rules.md`

## 必须遵循的项目模式

### 页面级

- 页面入口先判断访问权限
- 无权限时优先复用企业无权限组件或统一降级视图
- 不要在页面各处分散重复权限判断

### 按钮级

- 新增、编辑、删除、导出等按钮按权限显隐或禁用
- 行内操作和批量操作分别处理
- 不要只做前端控制，接口层仍需后端兜底

### 数据级

- 查看模式、编辑模式、审批模式可按权限切换字段可编辑状态
- 列配置、搜索项、操作列都可以基于权限裁剪

## 实现清单

1. 明确页面权限点和按钮权限点
2. 在页面初始化阶段完成权限判断
3. 为按钮、操作列、表单项补充权限控制
4. 权限不足时给出统一空状态、禁用态或无权限提示

## 何时切换到其他技能

- 当页面主体其实是标准列表页，而权限控制只是其中一层附加约束时，切换到 `basic-list-page`
- 当页面主体是树节点驱动的联动结构，而权限控制只是附加条件时，切换到 `tree-list-page`
- 当主要复杂度已经转向弹窗状态流和表单行为，而不是权限判断本身时，切换到 `dialog-skill` 或 `form-skill`
- 当阻塞点变成审批动作与权限联动，而不是一般页面权限时，补充 `approval-workflow-skill`

## 注意事项

- 前端权限控制只提升体验，不替代后端鉴权
- 权限字段命名必须与企业权限中心一致
- 权限规则优先抽成可复用函数，不要散落在模板里
