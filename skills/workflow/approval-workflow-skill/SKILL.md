# 审批流技能
用于生成企业审批流相关页面和操作区。

## 适用场景

- 审批单详情页
- 审批操作弹窗
- 审批意见填写
- 流程节点和审批记录展示
- 退回、通过、驳回等动作处理

## 按需读取的参考文件

- 审批页模板：`references/templates.md`
- 审批规则：`references/rules.md`

## 必须遵循的项目模式

### 页面

- 审批详情与审批操作分区展示
- 审批记录、节点状态、业务数据分层展示
- 操作按钮按审批状态和权限控制

### 数据

- 详情数据、流程数据、审批记录数据分别管理
- 审批提交动作独立封装

### 交互

- 通过、驳回、退回等动作统一走确认和反馈
- 审批意见校验独立处理

## 实现清单

1. 拆分业务详情区和审批流程区
2. 定义审批状态、动作和权限规则
3. 实现审批意见输入、动作提交、结果反馈
4. 展示审批记录和节点状态

## 何时切换到其他技能

- 当页面主体其实是详情展示，而审批流只是附加区域时，切换到 `detail-page-skill`
- 当按钮和字段权限控制本身更复杂，已经超过审批动作编排时，切换到 `permission-page`
- 当当前任务的主要复杂度在提交反馈、确认文案和异常提示，而不是审批状态机时，切换到 `message-skill`
- 当阻塞点变成审批接口契约、动作参数和流转结果结构时，补充 `api-skill`

## 注意事项

- 审批状态分支要集中管理
- 审批意见、驳回原因等字段要有最小校验
- 审批动作必须明确不可重复提交保护
- 生成审批流代码时补充中文注释，重点说明状态机分支、权限判断、动作提交流程和重复提交保护
