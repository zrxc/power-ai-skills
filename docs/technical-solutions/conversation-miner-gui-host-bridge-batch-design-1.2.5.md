# conversation-miner gui host bridge batch design 1.2.5

## Goal

`1.2.5` expands the already validated GUI-host bridge sample pattern from:

- `Trae`
- `Cursor`

to the remaining GUI / IDE style tools that can usually provide the final confirmed response text but may not be able to run the CLI capture command directly:

- `Windsurf`
- `Cline`
- `GitHub Copilot`

## Delivered Assets

`1.2.5` adds:

```text
.power-ai/adapters/windsurf-host-bridge.example.ps1
.power-ai/adapters/cline-host-bridge.example.ps1
.power-ai/adapters/github-copilot-host-bridge.example.ps1
```

All three scripts accept the same input modes:

- `-ResponsePath`
- `-ResponseText`
- `-UseClipboard`
- `-KeepTempFile`

Each script writes a tool-specific temp file and then delegates to:

```powershell
.\.power-ai\adapters\<tool>-capture.example.ps1 -ResponsePath <temp-file> -QueueResponse -ConsumeNow
```

## Design Notes

- The host-bridge builder introduced in `1.2.4` is reused directly.
- No new conversation-miner runtime behavior is introduced in `1.2.5`.
- This version is intentionally a rollout release:
  - same bridge contract
  - more concrete host samples
  - more `doctor` checks
  - more end-to-end proof

## Validation

`1.2.5` is complete when:

- the 3 new bridge scripts are generated by `init` / `sync`
- `doctor` checks all 3 bridge assets
- there is one end-to-end host bridge test for each new tool
- generated docs and release notes expose the new scripts
